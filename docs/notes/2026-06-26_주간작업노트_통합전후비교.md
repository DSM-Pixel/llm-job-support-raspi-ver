## 2. 통합으로 바뀐 것 (6/22)

가장 큰 변화는 **"3개의 따로 도는 데모"를 "한 화면에서 6개 메뉴로 도는 업무 플랫폼"으로 묶은 것**.

- **신규 `web/`**: 대시보드 / 데이터관리 / 이미지 라벨링 / 자연어 질의 / RAG 검색 / 요약보고서
  6개 페이지 (HTML 6 + CSS 7 + JS 7, 약 9,000줄)
- **신규 `backend/`**: FastAPI가 `web/` 정적 서빙 + `/api/*` 제공.
  **입출력 계약(스키마)을 먼저 고정**하고 내부는 MOCK으로 채워,
  프로토타입의 실제 로직을 나중에 갈아끼우는 구조로 설계.
- 프론트의 `setTimeout` 하드코딩 가짜응답 → `fetch` 호출로 전부 교체.
  `common.js`에 `api()` / `escapeHtml()` 공용 헬퍼 추가.

> **핵심 설계 결정**: "통합 = 기능 완성"이 아니라
> "통합 = 껍데기와 계약을 먼저 맞추고, 그 다음 속을 실제로 채운다".
> 이게 이번 주 작업 전체의 뼈대였다.

## 3. 통합 후 채워 넣은 실제 기능 (기능별)

### 이미지 라벨링
- MOCK → 실제 YOLO 탐지 (`backend/yolo_service.py` 신설)로 교체
- 박스 직접 그리기 모드, 다중 이미지·폴더 단위 라벨링
- 모달 내 사진 네비게이션, "폴더 전체 AI 라벨링" 버튼
- 라벨된 이미지 저장/다운로드

### RAG 검색
- 질의-문서 연관도(0~100점), 업로드/웹문서 색인
- **근거가 없으면 억지로 답하지 않고 '근거 없음'으로 응답**
- 참고 파일 클릭 시 원문 열람, 전체 초기화

### 자연어 질의
- 질문을 분류해서 웹 답변 / 데이터 검색으로 분기
- 화면 컨텍스트 기반 AI 대화 패널 (본문 옆에 나란히), 대화 영속

### 요약보고서
- Gemini 웹 그라운딩 기반 실데이터 생성
- "내 작업에서 가져오기"로 분석·라벨 이미지·RAG 결과 삽입
- 사진 첨부, AI 대화로 본문 수정, 활동 기반 통계 보고서, PDF 출력

### 대시보드
- 가짜 지표 → **실제 Gemini 토큰 사용량·한도(분당/하루) 퍼센트 막대**
- 실시간 폴링, 한도 소진 시 재사용 가능 시각, '마지막 성공' 시각, 주간 처리량 차트

### 공통 UX
- 다크/라이트 모드, 브랜딩(로고·파비콘·차분한 팔레트)
- 기록 관리 메뉴, 데이터 관리 실데이터 연동
