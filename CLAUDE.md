# llm-job-support

지엔소프트(주) × 유클리드소프트 「프로젝트형 청년 일경험」 멀티모달 AI 프로젝트 워크스페이스.
참여 청년이 **Claude Code 바이브 코딩** 방식으로 멀티모달 Vision AI / 생성형 AI 서비스의
**기획안 + 기능별 프로토타입**을 만들어내는 것이 최종 목표입니다.

> 본 CLAUDE.md는 모든 세션에서 자동으로 로드되는 프로젝트 컨텍스트입니다.
> 팀별 세부 컨텍스트는 `teams/<team-name>/CLAUDE.md`에 따로 둡니다(있을 경우).

---

## 프로젝트 한 줄 요약

VLM·SAM·YOLOe·LLM·Fine-tuning·Hybrid RAG·AI Agent 기술을 결합해
**자연어 질의 → 이미지 이해/검색/요약/보고서 생성/업무 자동화**를
수행하는 AI 서비스 후보 모델을 발굴·검증한다.

## 핵심 일정 (2026년)

- 수행기간: **2026-06-04 ~ 2026-07-29 (8주)**
- 1회 코칭: 프로젝트 개요/문제 공유, 데이터 수집/결과 정리
- 2회 코칭: 결과물 + 발표 피드백
- 최종 산출물: 기획 보고서 + 기능별 프로토타입 + 발표자료

## 팀 구성

- 총 5팀 × 4명 = 20명 (참여 청년)
- 각 팀은 세부 과제 1개를 선택하여 진행
- 가능 세부 과제 예: 도로 파손 라벨링 자동화, CCTV 이상행동 검색·분석,
  시설물 점검 데이터 운영, 공공데이터포털 연계 QA, 하이브리드 RAG 지식검색,
  요약/보고서 자동화 등

## 기술 스택

- **언어**: Python (필수). 프론트가 필요하면 **Gradio**를 1순위로,
  복잡하면 FastAPI + 간단한 HTML/JS.
- **패키지 매니저**: `uv` (속도/재현성). `pip`도 동작은 함.
- **포맷/린트**: `ruff` (format + lint 통합). 별도 black/flake8 추가 금지.
- **테스트**: `pytest` (필요 시).
- **모델/라이브러리 후보**:
  - VLM: Qwen2.5-VL, LLaVA, InternVL 계열
  - 분할: SAM / SAM2, YOLOe
  - LLM: Claude API (Anthropic SDK), 로컬 추론은 vLLM/Ollama
  - RAG: LangChain or LlamaIndex (얇게), ChromaDB/FAISS, BM25 하이브리드
  - Agent: Anthropic SDK tool-use, MCP

## 바이브 코딩 원칙

이 프로젝트는 “정답 코드 한 번에 작성”이 아니라 **빠른 반복 구현 + 시연**이 목적입니다.

- **시연 가능한 최소 결과물**을 먼저 만들고 거기서 개선한다.
  - Gradio `gr.Interface` 한 줄로 띄울 수 있으면 그렇게 한다.
  - 추상화·클래스화는 같은 패턴이 3번 이상 반복된 뒤에.
- **데모 시나리오 우선**:
  - "포트홀 영역을 찾아줘", "공공데이터로 통계 보여줘",
    "검색 결과를 보고서로 만들어줘" 같은 한 줄 시나리오가 돌아가야 한다.
- **무거운 정식 모델 학습은 마지막에**. 처음에는 사전학습 모델 + 프롬프트로 시작.
- **MOCK 데이터로 흐름을 먼저 잡고** 실제 데이터·API는 나중에 붙인다.

## 코딩 컨벤션

- **명시적 > 마법**: 매직 메서드, 메타클래스, 동적 import 자제.
- **함수 우선**: 클래스는 상태가 진짜 필요할 때만.
- **타입 힌트는 공개 함수에만**. 내부 헬퍼는 생략 OK.
- **에러는 사용자에게 의미 있게**. Gradio/Streamlit UI는 raise보다
  `gr.Error("이미지를 먼저 업로드해주세요")` 형태가 낫다.
- **시크릿/API 키는 `.env`** (절대 코드에 하드코딩 X, 절대 커밋 X).
- **노트북(.ipynb)은 실험용**, 발표/시연 산출물은 `.py` 또는 Gradio 앱으로 정리.
- **한글 주석 OK**. 단, 함수/변수명은 영어.

## 협업 / 깃 정책

- **기본은 `main`에 직접 commit/push OK**. 특별한 지시가 있을 때만 브랜치(`team-<n>/<topic>`)·PR을 쓴다.
- 커밋 메시지는 한국어 OK. 단 동사로 시작:
  `포트홀 라벨링 데모 추가`, `RAG 검색 결과 요약 기능 수정` 등.
- 큰 모델 weight, 데이터셋 원본은 **커밋 금지**. `data/`는 기본 ignore.

## Claude Code 활용 가이드 (팀원에게)

1. **막막하면 `/run`** — 만든 코드/Gradio 앱을 바로 띄워본다.
2. **기획서 쓸 땐 `/planning-report`** — 표준 양식 기반으로 작성.
3. **새 팀 시작 시 `/team-init <team-name>`** — 팀 워크스페이스 자동 생성.
4. **VLM/RAG/공공데이터 질문은 전용 subagent 사용** — `.claude/agents/` 참고.
5. **모르는 라이브러리는 그냥 물어봐도 됨**. Claude가 설치 명령까지 알려준다.

## 장기 기억 (memory MCP)

세션이 바뀌어도 진행상황·결정사항을 이어가기 위해 **`memory` MCP 서버**를 쓴다.
(`.claude/mcp/memory_server.py`, stdlib만, SQLite `.claude/memory.db` — 자동 로드 안 되므로 평소 토큰 0.)

- **세션 시작 / 맥락이 필요할 때**: 먼저 `memory_search`로 관련 기억을 꺼내본다
  (예: "지난번 포트홀 데모 어디까지 했지").
- **중요한 사실·결정·진행상황이 생기면**: `memory_save(content, tags)`로 바로 저장한다.
  한 건 = 한 사실, 태그는 공백 구분(예: `prototype api-test 진행상황`).
- 도구: `memory_save` / `memory_search` / `memory_recent` / `memory_delete`.
- DB는 커밋 금지(`.gitignore` 처리됨). 팀 공유가 필요한 합의사항은 이 CLAUDE.md나 `docs/`에 적는다.
