# 공공데이터 연동 사전 체크리스트

`/public-data-rag` 시작 전에 아래를 순서대로 확인한다. 하나라도 미확인이면 사용자에게 알리고 진행 여부를 묻는다.

## 1. 인증키 발급

- [ ] data.go.kr 회원가입 + 해당 API **활용신청** 완료 (자동승인 / 심의승인 구분 — 심의는 며칠 걸림)
- [ ] 키 형태 확인: **Decoded 키 권장** — Encoded 키를 requests `params`에 넣으면 이중 인코딩으로 `SERVICE_KEY_IS_NOT_REGISTERED_ERROR`가 난다
- [ ] 키는 `.env`의 `DATA_GO_KR_KEY`에만 저장 (코드·커밋 금지)
- [ ] 활용신청 직후에는 키 반영까지 최대 1시간 지연 가능

## 2. 트래픽 제한

- [ ] 일일 호출 한도 확인 (개발계정 기본 **1,000회/일**인 API가 많음 — 운영계정 신청 시 증설)
- [ ] 페이지네이션 파라미터(`pageNo`/`numOfRows`) 확인 — 전체 수집에 몇 회 필요한지 먼저 계산
- [ ] 수집 결과는 `backend/storage/pubdata.db`에 캐싱 → 개발 중 반복 실행이 한도를 까먹지 않게
- [ ] 429/`LIMITED_NUMBER_OF_SERVICE_REQUESTS_EXCEEDS_ERROR` 시 백오프 + 캐시 폴백 동작 확인

## 3. 라이선스 (공공누리)

- [ ] 데이터셋 상세페이지의 **공공누리 유형(1~4유형)** 확인:
  - 1유형: 출처표시 — 상업적 이용·변경 가능 (가장 자유로움)
  - 2유형: 출처표시 + **상업적 이용 금지**
  - 3유형: 출처표시 + **변경 금지**
  - 4유형: 출처표시 + 상업적 이용 금지 + 변경 금지 (가장 제한적)
- [ ] 오픈소스 대회 출품·시연에 쓸 수 있는 유형인지 판단 (2·4유형은 상업 활용 불가 — 출품 요강과 대조)
- [ ] 답변/보고서에 **출처표시 문구** 자동 포함되는지 (기관명 + 데이터셋명)
- [ ] 리포 라이선스는 AGPL-3.0 고정 — 데이터 라이선스와 별개지만, 함께 배포하는 샘플 데이터가 재배포 가능한지 확인

## 4. 데이터 자체

- [ ] 응답 포맷 JSON/XML 중 무엇인지 (둘 다 되면 JSON)
- [ ] 첫 1건을 `json.dumps(..., indent=2, ensure_ascii=False)`로 찍어 실제 스키마 확인 (문서와 다른 경우 흔함)
- [ ] 갱신 주기 (실시간/일간/연간) — 캐시 TTL 결정에 사용
- [ ] **개인정보**(차량번호, 얼굴, 주소 상세 등) 포함 여부 — 있으면 마스킹/제외 후 인덱싱
- [ ] 좌표계(위경도 vs TM), 인코딩(간혹 EUC-KR) 같은 함정 확인
